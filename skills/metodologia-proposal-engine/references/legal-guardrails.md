# Legal Guardrails — MetodologIA Proposal Engine
# Protects MetodologIA and its commercial team from:
# (1) over-promising undeliverable outcomes
# (2) legal exposure from unsupported claims
# (3) price/scope inconsistencies between proposal and canonical
# Without sacrificing: commercial appeal, differentiators, Minto structure

## 1. Hard Blockers — fix before generating any file

| ID | Trigger | Fix |
|----|---------|-----|
| L1 | Price ≠ current canonical | Replace with canonical price. Never adjust without canonical update. |
| L2 | [POR CONFIRMAR] item stated as confirmed | Add explicit conditional (see §6) |
| L3 | Guarantee clause differs from exact wording | Replace with exact clause (see §6) |
| L4 | Workshop→Bootcamp credit terms differ from "100%, 6 months, non-transferable" | Replace exactly |
| L5 | Result % stated as a promise (not target) | Wrap: "Indicative target: X. Actual result depends on adoption consistency. MetodologIA does not guarantee specific figures." |
| L6 | Scope includes something in canonical's "What it is NOT" section | Remove; or go to Innovation Mode and mark [POR CONFIRMAR] |
| L7 | Red list word present (see §4) | Replace with approved alternative |
| L8 | Unconfirmed credit chain stated as fact | Replace with "Credit applicable subject to current policy — ask your ambassador." |
| L9 | IAC service with B2C price (not confirmed) | Mark [POR CONFIRMAR]; do not show price |
| L10 | Fixed USD exchange rate used | Add "(indicative rate, subject to variation)" |

**Assumption [low risk]:** L3 applies to all guarantee mentions, including paraphrases. Even "money-back guarantee" without conditions is a violation — it omits the 4-hour limit and the feedback requirement.

## 2. Warnings — flag to user but do not block generation

| ID | Trigger | What to flag |
|----|---------|-------------|
| W1 | Co-branding or white-label mentioned | "Requires signed master contract + brand use agreement before first execution." |
| W2 | Group >20 participants | "Add co-facilitation (+50% base price) or double execution (-20% second run)." |
| W3 | In-person (presencial) mentioned | "Logistics and travel costs quoted separately." |
| W4 | Specific AI engine mentioned | "Must be validated with client's IT at least 3 business days before start." |
| W5 | Competence certificate mentioned | "Requires final project approved by rubric. Attendance certificate is issued to all." |
| W6 | LMS / Campus mentioned | "Included: during training + 1 month post-close. Continuation: COP 20,000/month." |
| W7 | Innovation Mode active | "Include disclaimer: this is a custom service design proposal; prices marked [POR CONFIRMAR] are indicative until SOW." |

## 3. Safe Phrasing Bank

These phrases are commercially compelling and legally sound. Use them freely.

```
"Designed so that adoption is the path of least resistance."
"Participants build assets that belong to them — no vendor dependency."
"The outcome is produced by the team; MetodologIA installs the system to produce it."
"We don't promise specific hours. We promise the method to find them."
"A single workflow saved justifies the full program cost."
"Built on evidence, not demos. Every module closes with a verifiable deliverable."
"Human-in-the-loop: the AI proposes; the team decides. Always."
"The system stays in the team — not in the consultant."
"Methodology access without permanent provider dependency."
```

**For result claims — always use this wrapper:**
```
"Indicative target: [result]. Participants who apply the system consistently 
report [result]. Actual outcome depends on adoption level. 
MetodologIA does not guarantee specific figures."
```

## 4. Red List

### Global — zero tolerance in any document
```
hack · truco (trick) · secreto (secret) · resultados instantáneos / instant results
arquitecto / architect · arquitectura / architecture  [in methodological context]
transformación / transformation  → replace with (R)Evolución / (R)Evolution
revolucionario / revolutionary · disruptivo / disruptive
único en el mercado / unique in the market
soluciones innovadoras / innovative solutions
equipo de expertos / team of experts
resultados garantizados / guaranteed results
sin riesgo / zero risk
```

### Per-audience additions

**Novice user / inexperto:** ROI, OKR, KPI, SLA, framework, agile methodology, sprint, stakeholder, deliverable, pipeline, human-in-the-loop, SSOT, DoD, RACI, capstone, meta-prompt, co-creation [see substitution table in `catalog/segments.yaml`]

**Procurement / Compras:** innovador, revolucionario, único, líder (and EN equivalents) — no marketing adjectives whatsoever

**Executive (both):** SLA exact values, checklist details, campus/LMS operational details

## 7. Pre-Delivery Checklist

Run before generating any file. If any L blocker fails → fix and rerun.

```
HARD BLOCKERS (all must pass):
□ L1: All prices match current canonical
□ L2: No [POR CONFIRMAR] item stated as confirmed
□ L3: Guarantee clause = exact wording (§6)
□ L4: Workshop→Bootcamp credit = exact terms (§6)
□ L5: All result % wrapped with "indicative target"
□ L6: Nothing from canonical "What it is NOT" in scope
□ L7: Red list (global) = 0 occurrences
□ L8: All unconfirmed credits have conditional language
□ L9: No IAC B2C price without [POR CONFIRMAR]
□ L10: All USD figures marked "(indicative rate)"

WARNINGS (check and note active ones):
□ W1–W7: reviewed, active warnings noted in verification report

TONE + BRAND:
□ Per-audience red list = 0 occurrences
□ Novice substitutions applied where audience = inexperto
□ "(R)Evolution" / "(R)Evolución" used, never "transformation"
□ Minto: conclusion + ≥2 MECE supports + evidence + 1-step CTA
□ IVA discriminated (B2B); included (B2C online)
□ Brand mode applied: tokens resolved correctly
□ Whitelabel: ZERO Metodología references in output files
□ Cobrand: ONLY gold token overridden by partner primary
```
