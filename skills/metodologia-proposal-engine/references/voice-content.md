# Use when drafting proposal content in Layer 2.

## Activation line
✅ ACTIVE: MetodologIA Brand Voice v3.0 (Minto-First OS)

## Golden rule
*If a sentence doesn't help someone decide, understand, or act — it doesn't belong.*

---

## 1. Minto-First Structure

### Complete (default — use when: trade-offs, high impact, multiple stakeholders)
```
CONCLUSION       → Decisory recommendation in 1–2 sentences.
                   TEST: Does it answer "What do I do now?" without reading further?

SUPPORT 1 [P1]   → (R)Evolution pillar: current → desired state gap closed by method.
                   Typical: diagnosis, system design, cadence.

SUPPORT 2 [P2]   → Intention-before-intensity pillar: design over force.
                   Typical: focus, explicit trade-offs, friction reduction, sustainable cadence.

SUPPORT 3 [P3]   → Technology-as-ally pillar: automate the repeatable; amplify the important.
                   Typical: templates, reusable assets, automation of measurement.

EVIDENCE         → One per support: real data | suggested indicator | measurable signal | required data.
                   If data is missing: declare "Required data: [what]" — never invent.

CTA              → Verb + object + context (when/where/who).
                   TEST: Does it have a concrete verb? Is it one step? Executable today/this week?
```

### Micro (use when: chat, limited attention, low stakes)
```
CONCLUSION + 2 MECE SUPPORTS + EVIDENCE + CTA
```

### MECE tests for supports
- **Exclusion:** Are two supports reformulations of the same point? If yes → merge.
- **Exhaustivity:** If you remove one support, does the conclusion lose its foundation? If yes → it's necessary.
- **Non-fusable:** Can you merge two without losing meaning? If yes → not MECE.

## 2. Pillars — Conceptual Engine

| Pillar | Concept | Application in proposals |
|--------|---------|--------------------------|
| P1 | **(R)Evolution**: structural change + operational continuity | Gap analysis, method to close it, phased approach |
| P2 | **Intention before intensity**: design > force | Prioritization, what not to do, path of least resistance |
| P3 | **Technology as ally**: automate repeatable; amplify important | Reusable library, playbooks, workflow assets |

**Rule:** if a support doesn't anchor to a pillar → it's decoration, not a support.

## 4. Evidence Taxonomy

| Type | When to use | Format |
|------|-------------|--------|
| **Real data** | Comes from user/context in conversation | Cite source in brackets |
| **Suggested indicator** | Metric the client could measure | `Indicator: [name] | Frequency: [x] | Owner: [role] | Threshold: [what = good]` |
| **Measurable signal** | Observable proxy for outcome | `Signal: [behavior or event that indicates progress]` |
| **Required data** | Input needed to substantiate a claim | `Required to substantiate: [specific data point]` |

**Minimum evidence rule:** every strong claim (with numbers, %, or impact language) must use one of these four types.

## 5. CTA Quality Standards

**Passes:**
- Has action verb + specific object + context (date/channel/who)
- Executable in one step
- Specific enough to act on today

**Fails:**
- Generic: "improve", "align", "move forward"
- No concrete verb
- Requires multiple steps before action

**Approved CTA templates:**
```
"Schedule a 15-minute call this week: [link/channel]"
"Write to [email] with subject '[Company] — Proposal'"
"Confirm the scope by replying to this message before [date]"
"Send the completed brief to your ambassador before [date]"
```

## 6. Copywriting Patterns (use when they earn their place)

| Pattern | When | Limit |
|---------|------|-------|
| Empathetic question | When client tension is diffuse | 1 per document |
| Honest comparison | When client is evaluating alternatives | Always honest — no fabricated advantages |
| "Not for you if..." | To filter anti-fit and build credibility | Subtle section, not aggressive |
| Testimonial frame | To illustrate outcome | Only with real testimonial; use empty frame otherwise |

**Rule:** if the pattern doesn't improve decision, understanding, or action → remove it.

## 8. Quality Rubric /20 (Audit Mode only — use when user requests)

2 points each (0=fail, 1=partial, 2=meets):

| # | Criterion |
|---|-----------|
| 1 | **Foundation**: clear causality between conclusion and supports |
| 2 | **Veracity**: evidence per support; no unsupported strong claims |
| 3 | **Quality**: no contradictions or critical ambiguities |
| 4 | **Useful density**: every sentence aids decision/understanding/action |
| 5 | **Simplicity**: minimum necessary without sacrificing precision |
| 6 | **Clarity**: terms defined where they matter |
| 7 | **Precision**: conditions and limits explicit |
| 8 | **Depth**: causes and trade-offs surfaced, not just recommendations |
| 9 | **Coherence**: structure and voice consistent throughout |
| 10 | **Value**: actionable today + progress measurable this week |

Pass: 20/20 for v1 releases · ≥16/20 for internal drafts.
